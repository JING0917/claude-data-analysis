## 2. 现有促活体系诊断

### 2.1 四大机制参与数据

| 机制 | 每日参与 | 占DAU | 状态 |
|------|---------|--------|------|
| 签到 | 20万 | 17.4% | 主力，转化率极高 |
| 幸运抽奖 | 4.5万 | 3.9% | 第二入口，90%+奖品为饭票 |
| 红包雨 | 1万 | 0.9% | 几乎未打穿，时段骨架可用 |
| 下单挑战赛 | 1万 | 0.9% | 小众精准，非常态运营 |

签到与抽奖重合 3.3万，去重后激励体系触达约 **21.2万人（DAU的18.4%）**。

### 2.2 签到漏斗（关键发现）

```
签到 20万
  ├── 报名（下单）15.7万  → 签到→下单转化率 78.5%
  │     └── 完单 14.6万   → 报名→完单 93%
  └── 未下单 4.3万        → 21.5%
```

**签到用户的下单转化率 78.5%，远高于整体 DAU 平均水平。** 签到一旦形成习惯，下单几乎是必然行为。

### 2.3 核心判断

| 问题 | 结论 |
|------|------|
| 81.6% 的DAU不碰任何激励 | 机制触达面窄，不是机制不够好 |
| 签到转化率 78.5% | 签到是验证过的最强转化杠杆 |
| 中低频 35.2万 DAU大概率未被覆盖 | 增量空间在「让更多人开始签到」 |

**策略方向：不是优化签到奖励，也不是新增机制，而是把签到入口铺到用户必经路径上，扩大签到覆盖率。**

## 3. 方案设计

### 3.1 核心逻辑

```
现状：用户来→找活动→下单→走人（全程未接触签到）
目标：用户来→找活动→下单→「下单后引导签到」→次日回来签到→再下单
```

**签到→下单转化已验明 78.5%，杠杆点是签到人数，不是签到奖励。**

### 3.2 三层改动

#### 第一层：下单完成页 — 签到预约入口（P0）

| 元素 | 设计 |
|------|------|
| 位置 | 下单完成页，订单状态上方 |
| 文案 | 「明日签到，多领 X 倍元宝」 |
| 交互 | 一键预约明日签到提醒 |
| 逻辑 | 今下单 → 明日签到奖励翻倍 → 引导连续行为 |

**这是整个方案的核心改动。** 每天有下单行为的用户远大于 20 万签到用户，在下单完成页埋入口，用「明日签到翻倍」做钩子，把下单用户往签到体系导。

#### 第二层：Push 触发体系（P1）

| 时机 | 触发条件 | 文案方向 |
|------|---------|---------|
| 早 9:00 | 昨天下单但未签到 | 「昨天下了单，今天签到多领 5 倍元宝」 |
| 午 12:00 | 今天已签到但未下单 | 「签到元宝已到账，来看看今天有什么活动」 |
| 晚 19:00 | 今日未签到 | 「今天还没签到，元宝要过期了」 |

**关键：Push 文案不说「来签到」，说「昨天下单了今天签到更值钱」。** 把签到的价值锚定在昨天的下单行为上，而不是凭空喊人来签到。

#### 第三层：签到卡片升级 — 进度可视化（P2）

| 现有 | 改为 |
|------|------|
| 签到得固定元宝 | 昨天下单 → 今日签到 5 倍元宝；昨天没下单 → 基础元宝 |
| 签到完即结束 | 签到后展示：「连续签到 X 天，再签 Y 天得 2 元红包」 |
| 单独签到页 | 签到后底部推荐活动，引导下单 |

## 5. 预期效果

### 5.1 签到覆盖率推演

| 场景 | 签到覆盖DAU | 增量下单（按78.5%转化） |
|------|------------|----------------------|
| 当前 | 20万 | 15.7万 |
| 保守（签到渗透率从17.4%→25%） | 29万 | +7.1万 |
| 中性（→30%） | 35万 | +11.8万 |
| 乐观（→35%） | 40万 | +15.7万 |

### 5.2 DAU 增量路径

```
促活（签到杠杆扩大）   → +8~12万 DAU
  +
MAU 增长（渠道放量）   → 填补剩余缺口
  +
防流失（签到粘性提升） → 减少存量流失，间接贡献
```

促活解决的是「来的人多来」，能贡献 10 万左右的 DAU 增量。剩余缺口需要 MAU 池子扩大 + 渠道拉新放量配合。

## 8. 风险与边界

- **转化率稀释**：签到覆盖从 20 万扩到 40 万后，转化率可能从 78.5% 下降到 50-60%，但绝对值仍增长。关注绝对值，不纠缠转化率。
- **红包雨改造谨慎**：6 场红包雨只有 1 万人参与，但时段骨架（10/11/12/14/16/19）匹配用户下单决策窗口。不要在 P0 阶段大改，先验证签到杠杆扩大有效。
- **不要新增机制**：四个机制已足够，81.6% 的用户不是缺新玩法，是根本没接触到现有玩法。先解决触达，再优化机制。
