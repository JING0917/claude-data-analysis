一、沟通管理

核心框架

过程要点
规划沟通管理基于干系人需求确定沟通方式、频率、内容；沟通渠道数 n(n−1)/2
管理沟通按计划收集、生成、分发信息；确保信息完整、及时、准确
监督沟通监控沟通效果，根据反馈调整；防止信息过载或遗漏

案例高频场景

  1. 干系人不满/投诉：信息传递不及时、沟通渠道不畅、干系人未识别到位
  2. 团队协作冲突：异地/虚拟团队沟通障碍、语言文化差异
  3. 需求变更引发的混乱：变更信息未同步所有相关方
  4. 汇报机制缺失：高层不了解项目状态，干系人信息不对称
答题框架 — 沟通问题通用四步法
  1. 诊断根因：定位是「规划缺失」「执行不到位」还是「监督反馈不足」
  2. 规划修复：更新沟通管理计划 → 明确干系人信息需求矩阵 → 确定沟通方式（推式/拉式/交互式）
  3. 执行改进：建立信息分发机制 → 选择合适的沟通技术 → 确保信息编码/解码一致
  4. 监督闭环：收集干系人反馈 → 评估沟通有效性 → 持续优化

记忆要点

  • 沟通渠道数公式：n(n−1)/2，新增1人增加 n 条渠道（原有人数）
  • 沟通方式三种：交互式（会议/电话）、推式（邮件/报告）、拉式（知识库/内网）
  • 沟通模型：发送方 → 编码 → 媒介 → 解码 → 接收方，全程有噪声干扰
  • 90%的项目经理时间用于沟通
  • 书面+口头的组合沟通效果最优

二、度量绩效域

核心框架（PMBOK 7th）

维度核心问题关键实践
制定度量指标度量什么？指标是否与目标对齐？SMART原则、KPI/KRI/OKR
收集数据数据来源可靠吗？收集成本合理吗？自动化采集、数据字典
分析数据偏差多大？趋势如何？挣值分析(EVM)、趋势分析、根本原因分析
做出决策基于数据，下一步做什么？偏差阈值触发纠正措施
沟通结果谁需要知道什么？何时？仪表盘、状态报告、评审会

案例高频场景

  1. 度量指标不合理：指标与项目目标脱节，只度量了容易度量的而非重要的（虚荣指标）
  2. 数据质量差：手工填报、口径不一致、数据延迟
  3. 只监不控：数据收集了但不分析、不决策，度量沦为形式
  4. 度量过度：指标太多导致团队负担重、注意力分散
答题框架 — 度量问题三步法
  1. 指标体系重构：重新对齐项目目标 → 定义领先指标（驱动型）和滞后指标（结果型）→ 设定合理阈值和预警线
  2. 数据质量保障：建立数据采集规范 → 自动采集替代手工 → 定期数据质量审计
  3. 度量驱动决策：建立评审节奏（如周度度量评审会）→ 偏差超过阈值自动触发纠正流程 → 度量结果可视化展示

记忆要点

  • 领先指标 vs 滞后指标：前者预测未来（如需求变更率），后者反映过去（如实际成本）
  • KPI vs KRI：KPI衡量绩效好不好，KRI衡量风险大不大
  • 度量陷阱：古德哈特定律——当一个度量变成目标时，它就不再是一个好的度量
  • 度量成本不应超过它所支持决策的价值

三、不确定性绩效域

核心框架（PMBOK 7th）

不确定性 ≠ 风险。不确定性有四种来源：

来源含义应对策略
风险 (Risk)已知-未知，可识别概率和影响规避/转移/减轻/接受
模糊性 (Ambiguity)需求或方案不清晰、有多种解读渐进明细、原型验证、多方案并行
复杂性 (Complexity)系统要素多、相互依赖、涌现行为解耦、模块化、系统思考
波动性 (Volatility)环境快速变化、难以预测敏捷迭代、预留缓冲、情景规划

案例高频场景

  1. 需求不确定：客户说不清要什么，需求频繁变更
  2. 技术不确定：采用新技术栈，团队不熟悉，可行性存疑
  3. 环境不确定：政策法规变动、市场变化、组织架构调整
  4. 复杂度失控：多系统集成、多供应商协调，牵一发动全身
答题框架 — 不确定性应对四维法
  1. 分类诊断：判断不确定性的主要来源（风险/模糊/复杂/波动）→ 不能一刀切
  2. 策略选择：风险→建立风险登记册；模糊→原型/试点；复杂→模块化解耦；波动→预留缓冲+滚动规划
  3. 组织保障：建立变更控制委员会(CCB) → 提高团队韧性 → 培养T型人才
  4. 过程适配：选择适合的生命周期（预测型/迭代型/增量型/敏捷型/混合型），不确定性越高越倾向敏捷

记忆要点

  • 韧性 (Resilience)：系统在扰动后恢复的能力，不只是"扛住"更是"弹回来"
  • 反脆弱 (Antifragile)：不仅承受波动，还在波动中变得更强——在项目中体现为持续学习和改进
  • 不确定性与生命周期选择直接关联：不确定性越高，越需要迭代和增量方式
  • 预留管理储备是关键实践——不是浪费，是对不确定性的理性投资

四、采购管理

核心框架

过程要点
规划采购管理自制/外购分析；合同类型选择（固定总价/成本补偿/工料）
实施采购招标/询价/谈判；卖方选择标准；采购工作说明书(SOW)
控制采购合同履行监控；变更管理；索赔管理；合同收尾

三种合同类型对比

合同类型买方风险适用场景
固定总价 (FFP)低范围明确、需求稳定
成本补偿 (CPFF/CPIF)高范围不明确、需灵活调整
工料合同 (T&M)中等工作量大但单价可定、范围中等清晰

案例高频场景

  1. 供应商选择不当：低价中标但交付质量差、供应商能力不足
  2. 合同条款模糊：验收标准不清、变更流程缺失、违约责任不明
  3. 采购过程不规范：规避招标、围标串标、未按规定审批
  4. 合同执行失控：供应商延期、超支、质量不达标，缺少有效监控
答题框架 — 采购问题通用对策
  1. 规划阶段：完整自制/外购分析 → 选择合适合同类型 → 编制详细的采购SOW → 制定选择标准（不唯低价论）
  2. 执行阶段：规范招标/询价流程 → 做好供应商尽职调查 → 合同条款明确（范围、验收标准、付款条件、违约条款、变更流程）
  3. 控制阶段：建立供应商绩效评审机制 → 定期对照合同条款核查 → 变更走正式变更控制流程 → 做好索赔和争议管理

记忆要点

  • 合同类型选择口诀：范围清→固定总价，范围模糊→成本补偿
  • 采购文档三大件：采购工作说明书(SOW)、询价文件(RFP/RFQ)、供方选择标准
  • 集中采购 vs 分散采购：前者降低成本但灵活性差，后者相反
  • 合同收尾四步：验收→结算→绩效评估→归档

五、风险管理

核心框架

过程要点
规划风险管理确定风险管理活动的方法、角色、预算、时间安排
识别风险头脑风暴、德尔菲法、SWOT、根本原因分析；输出风险登记册
定性风险分析评估概率和影响，生成概率影响矩阵，排出优先级
定量风险分析EMV（期望货币值）、蒙特卡洛模拟、敏感性分析、决策树
规划风险应对威胁→规避/转移/减轻/接受；机会→开拓/分享/提高/接受
实施风险应对将应对措施转化为具体行动，分配责任人
监督风险风险审计、储备分析、风险评审会；关注次生风险

案例高频场景

  1. 风险管理缺位：没有风险登记册、没有定期风险评审，出了问题临时应对
  2. 风险识别不全面：只关注技术风险，忽略了管理风险、外部风险、干系人风险
  3. 应对措施空洞："加强监控""注意防范"——没有具体的责任人、时间、触发条件
  4. 忽视次生风险：应对措施本身带来新问题（如赶工导致质量下降）
答题框架 — 风险管理缺陷通用对策
  1. 建章立制：制定风险管理计划 → 明确角色职责（风险负责人）→ 设定风险评审节奏
  2. 全面识别：多维度识别（技术/管理/外部/组织）→ 多方法组合（头脑风暴+检查表+假设分析）→ 更新风险登记册
  3. 科学分析：定性：概率影响矩阵 → 定量：EMV/决策树（涉及金额时用定量）→ 确定风险优先级
  4. 落地应对：每个高风险有具体应对预案 → 明确触发条件和责任人 → 预留应急储备 → 跟踪次生风险

记忆要点

  • 风险应对策略（威胁）：规避（消除威胁根源）→ 转移（保险/外包，转移的是影响而非责任）→ 减轻（降低概率或影响）→ 接受（被动接受或预留储备主动接受）
  • EMV = 概率 × 影响金额，用于比较不同应对方案的经济合理性
  • 应急储备 vs 管理储备：前者针对已知-未知风险（项目经理可控），后者针对未知-未知风险（管理层审批）
  • 风险登记册是活的文档，伴随项目全生命周期持续更新

EMV（预期货币价值）计算专题 — 已考！

公式：EMV = 概率 × 影响金额（收益为正，损失为负）

题型一：单风险EMV

某风险发生概率30%，发生损失50万。
EMV = 30% × (−50万) = −15万
说明：该风险的货币化期望损失为15万

题型二：机会（正收益）

采用新技术有40%概率节省工期，预计节省成本20万。
EMV = 40% × 20万 = +8万
积极风险（机会）EMV为正

题型三：多方案决策树（高频计算题）

场景：选择方案A（自研）还是方案B（外包）？

方案情景概率损益EMV计算
方案A：自研 成功60%+100万 60%×100 + 40%×(−30) = +48万
失败40%−30万
方案B：外包 顺利80%+50万 80%×50 + 20%×(−10) = +38万
延误20%−10万

决策：EMV(A)=48万 > EMV(B)=38万 → 选方案A自研

题型四：含决策树是否做测试/调研

先花5万做技术验证：

  • 验证通过（概率70%）→ 自研EMV = +60万
  • 验证不通过（概率30%）→ 外包EMV = +20万
  • 总EMV = −5 + (70%×60 + 30%×20) = −5 + 48 = +43万
  • 对比直接自研EMV = +48万 → 不测试直接做更划算（但风险更大）

答题注意事项

  • EMV只看期望值，不反映风险偏好——风险厌恶型组织可能选EMV低但方差小的方案
  • 计算时注意正负号：损失为负，收益为正
  • 所有分支概率之和必须= 100%
  • EMV用于定量风险分析过程，常见工具还有蒙特卡洛模拟、敏感性分析

案例答题通用模板 — 六步法

  1. 问题定位：从题干找出2-3个核心问题（对应过程缺失/执行不当/监督缺位）
  2. 理论对照：指出正确的理论做法（引用PMBOK过程或绩效域实践）
  3. 影响分析：说明该问题对项目目标（范围/进度/成本/质量）的具体影响
  4. 改进措施：给出3-5条具体可操作的改进建议（不能只说正确废话）
  5. 预防机制：提出如何避免同类问题在后续项目中再次出现
  6. 启示总结：1-2句话总结关键教训
