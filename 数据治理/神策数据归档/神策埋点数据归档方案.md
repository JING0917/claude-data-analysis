# 神策埋点数据归档方案

## 一、背景

神策私有化部署的埋点数据持续增长，磁盘占用过高。目前原始事件表（`ods.ods_sr_traffic_sensor_event_log_realtime`）日增约 **4亿条**，StarRocks 侧已独立存储，神策侧同样保留全量原始数据，两者叠加造成存储冗余。

- **最早数据日期**：2025-06-24
- **数据累积时长**：约 330 天（~11个月）
- **按90天归档后**：约 240 天数据转储COS，**预计释放约 73% 磁盘空间**

核心诉求：
- 降低神策集群磁盘使用量
- 历史数据可回溯查询（不要求高性能）
- 日常看板和告警不受影响

## 二、归档策略

### 2.1 保留周期

| 数据分层 | 时间范围 | 存储位置 | 查询性能 |
|----------|---------|---------|---------|
| 热数据 | 近 **90天** | 神策本地 SSD | 正常 |
| 冷数据（归档） | 90天以前 | 腾讯云 COS（S3协议） | 可用但较慢 |

### 2.2 归档触发

- **触发方式**：按天自动执行，每天凌晨将第91天前的数据转储到COS
- **粒度**：按日期分区归档，单日数据为一个归档单元
- **不可逆**：已归档日期不支持再导入新数据（神策限制）

### 2.3 存储后端

选用**腾讯云 COS**，通过 S3 兼容协议对接：

- 成本远低于本地 SSD 扩容
- 已有腾讯云账号，无需额外开通
- 支持生命周期管理（归档数据再过期后可自动删除）

## 三、影响评估

### 3.1 不受影响的场景

| 场景 | 原因 |
|------|------|
| 近30天PV/UV看板（红袖） | 数据在90天热窗口内 |
| 埋点质量监控（小时级） | 仅查近4小时+近8天环比 |
| 日常事件分析、漏斗分析 | 通常分析周期在90天内 |
| StarRocks侧所有查询 | StarRocks数据独立存储，与神策归档无关 |

### 3.2 受影响的场景（归档后变慢）

| 场景 | 影响程度 | 缓解措施 |
|------|:---:|------|
| 「上线至今」类指标 | ⚠️ 中 | 切换到近90天口径，或接受异步查询 |
| 年度趋势对比 | ⚠️ 中 | 提前预聚合到StarRocks，从SR侧出报告 |
| 偶尔的历史事件回溯 | 🔵 低 | 接受异步查询等待，一般几分钟内返回 |
| 用户行为序列（超90天前） | 🔵 低 | 低频场景，异步加载可接受 |

### 3.3 关键看板清单（需逐项确认）

归档前需要排查：神策中哪些看板/书签/概览的时间范围超过了90天？

## 四、实施步骤

### 阶段1：准备（1-2天）

1. 确认当前神策版本号，是否支持数据归档（需 ≥1.14）
2. 统计当前磁盘使用量和日增量
3. 统计当前数据最早日期，估算归档后可释放的空间
4. 在腾讯云COS创建专用Bucket，配置S3兼容访问密钥
5. 梳理受影响的看板清单（时间范围 > 90天的）

### 阶段2：与神策沟通（1次会议+跟进）

**沟通要点**：

1. 申请开启数据归档功能
2. 确认COS S3协议兼容性（AWS Signature版本）
3. 确认归档策略配置：按天归档，保留90天热数据
4. 要求提供归档后查询性能的基准测试数据
5. 确认归档数据删除流程（如需彻底清理）
6. 确认是否支持部分数据回热（紧急情况下恢复热数据）
7. 评估归档过程对线上查询的影响（网络带宽占用）

**提供给神策的信息**：

- 当前部署版本：
- 当前数据总量/日增量：
- 目标保留天数：90天
- 归档存储：腾讯云COS，S3兼容
- 预计开启时间：

### 阶段3：验证（3-5天）

1. 先在测试环境（如有）验证归档流程
2. 或选取少量历史日期先行归档，验证查询功能
3. 对关键看板逐项测试查询耗时，记录归档前后对比
4. 确认StarRocks侧数据同步不受影响

### 阶段4：正式执行（1天）

1. 业务侧发通知：X月X日开始归档，90天前数据查询会变慢
2. 开启归档功能，监控首日执行情况
3. 验证磁盘释放量是否符合预期
4. 关键看板确认可用

### 阶段5：持续优化

1. 监控归档任务每日执行状态
2. 记录归档数据查询频率，评估是否需要调整保留天数
3. 如磁盘再次紧张，可考虑缩短到60天或启用COS生命周期自动删除

## 五、风险评估

| 风险 | 概率 | 影响 | 应对 |
|------|:---:|------|------|
| 归档后关键看板查询超时 | 低 | 中 | 提前测试，受影响看板缩小默认时间范围 |
| COS S3兼容性问题 | 低 | 高 | 提前和神策确认，准备HDFS备选 |
| 归档过程中网络带宽打满 | 中 | 低 | 安排在业务低峰期（凌晨）执行 |
| 误归档后需要回热数据 | 低 | 中 | 确认神策是否支持回热，以及流程和耗时 |
| StarRocks同步受影响 | 极低 | 高 | SR数据链路与神策存储引擎独立，不受影响 |

## 六、预期收益

| 指标 | 预估 |
|------|------|
| 磁盘释放比例 | 从 2025-06-24 起累积约330天数据，归档90天前数据可释放约 **73%**（240天/330天） |
| 日常看板性能 | 不变（90天内数据仍在SSD） |
| 历史查询性能 | 从秒级降至分钟级（异步模式） |
| 存储成本 | COS成本约为SSD的1/5~1/10 |
